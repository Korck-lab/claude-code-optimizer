# Changelog

All notable changes to cc-optimizer are documented in this file.

## [1.0.0] - 2026-06-20

### Added

#### v2: Per-project MCP / plugin / connector cleanup
- **inventory.py** — analyzes exposed (Claude API connectors, globally-enabled plugins, MCP servers) vs. actual usage from session transcripts
- **apply2.py** — writes safe, reversible cleanups: `disableClaudeAiConnectors`, `enabledPlugins[id]=false`, removes dead `disabledMcpServers` keys
- **Worktree aggregation** — correctly handles inherited settings across worktrees and nested projects
- **Zero false positives** — verified against real plugin catalogs from `~/.claude/plugins/cache/`

### Enhanced

#### Token optimization (v1 baseline)
- **analyze.py** — deterministic session aggregation (raw-stats.json): models, tokens, cache metrics, output-size histograms
- **cost.py** — pricing-weighted analysis from fresh official docs (knowledge-base.json)
- **recommend.py** — rules engine generating applyable findings: model choice, subagent models, cache tuning, MCP bloat
- **apply.py** — safe config writer with deep-merge, per-file .bak, dry-run default, per-project settings.json mutations
- **briefs.py** — compact per-project summaries for optional LLM refinement
- **merge.py** — deterministic + LLM refinement merge
- **report.py** — findings → Markdown report

#### Documentation
- Complete pipeline documentation with examples
- Safety guarantees and validation procedures
- Usage patterns for zero-config analysis

### Technical

- **Python 3.9+** required
- **knowledge-base.json** — 182KB reference of Claude Code knobs and current official pricing (auto-regenerated by cc-cost-research workflow)
- **Zero model tokens** for core analysis (pure arithmetic on recorded usage data)
- **Optional LLM refinement** for top spenders (Opus 4.8 / Sonnet fallback)
